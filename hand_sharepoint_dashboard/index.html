What’s included
  • Progress: assessed vs not assessed + phone outcomes.
  • Active sites: totals for structures, households, individuals, latrines + structure mix.
  • Data quality: QC issue rate + QC-by-type.
  • Geo: boundary-driven filtering + choropleths; points where coordinates exist.
How to publish
  1. Deploy this folder to Vercel/Netlify/SharePoint static hosting.
  2. Replace assets/data/sites.json with the latest build output (or wire up /api/data).
  3. Optionally swap boundary files with TopoJSON later for performance.
Privacy by default

This build does not ship phone numbers or personal names inside client-side files.

If you need PII, keep it behind authentication and serve it from a server-side endpoint.
Troubleshooting
  • If the map looks blank: some sites have missing coordinates.
  • If polygons don’t match your dataset: regenerate the boundary subset or load full admin3.
